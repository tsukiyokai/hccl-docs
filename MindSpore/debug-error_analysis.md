# 报错分析

## 概述

本章节介绍MindSpore为神经网络开发者和框架开发者提供的多种功能调试能力。功能调试分为两类：

- 网络开发调试：为神经网络开发者提供，包括网络报错调试与网络执行调试
- 框架开发调试：为MindSpore框架开发者提供，例如日志、RDR等

## 网络报错调试

网络报错调试是解决网络训练或推理过程中出现的报错问题，包括报错分析和调试定位两部分。

### 报错分析

#### 信息归纳

获取以下几类信息：

1. 环境信息：操作系统、MindSpore版本、执行模式、设备信息
2. 错误描述信息：错误类型、错误描述、错误堆栈
3. 框架日志信息：怀疑是框架问题时需要获取

#### 报错信息结构

MindSpore报错信息采用Python Traceback处理，包括：

- Python堆栈信息：展示Python调用关系
- 报错类型和描述：指出具体的错误原因
- 与网络开发者相关的信息：包含"The Traceback of Net Construct Code"，映射C++后端编译报错到Python前端
- 与框架开发者相关的信息：包含C++堆栈，可通过设置`export MS_EXCEPTION_DISPLAY_LEVEL=1`隐藏

#### 错误分析步骤

1. 根据错误信息确认问题场景（数据加载、网络构建或分布式并行）
2. 进一步确认具体问题类型
3. 根据Python调用栈分析报错位置
4. 基于问题场景假设可能的根本原因

#### 错误搜索

- FAQ：按问题场景搜索常见报错问题
- 报错案例：查看典型报错案例及解决方法
- 社区Issue：搜索开源社区反馈的问题

### 调试定位

#### 策略选择

- 静转动调试：使用`set_context(mode=mindspore.PYNATIVE_MODE)`切换到动态图模式，便于逐行调试

- 异步转同步调试：使用`set_context(mode=mindspore.PYNATIVE_MODE, pynative_synchronize=True)`切换同步执行

- 二分法策略：简化问题场景，通过分别调试不同模块验证假设

- 演绎推理策略：根据问题因果链条逐步反向推理，定位根本原因

#### 问题复现

问题稳定复现是调试前提，需要处理随机初始化和不同输入数据导致的随机性。

#### 调试验证

动态图调试：

```python
import pdb; pdb.set_trace()
```

在代码前插入，使用pdb进行单步调试和过程跟踪。

静态图调试：

- 使用`ops.print_`接口打印Tensor或字符串信息
- 使用Debugger调试工具查看图节点输出、设置监测点、追踪训练异常

## 网络执行调试

### 执行观测

| 功能             | 说明                                           |
| ---------------- | ---------------------------------------------- |
| 训练过程可视化   | 记录标量、图像、计算图等信息                   |
| metrics          | 评估训练结果，提供accuracy、loss、precision等指标 |
| print_接口       | 打印Tensor或字符串信息                         |
| 中间文件保存     | 保存IR文件用于图结构诊断                       |
| 数据Dump         | 保存算子输入输出数据进行调试                   |

### 执行控制

| 功能     | 说明                                                 |
| -------- | ---------------------------------------------------- |
| Callback | 在特定时期执行特定动作，如保存模型、监控loss         |
| Hook     | 在pynative模式捕获中间层算子输入输出和反向梯度       |
| 同步执行 | 控制算子在device上是否使用异步执行                   |

## 框架开发调试

| 功能分类 | 主要功能 | 说明                               |
| -------- | -------- | ---------------------------------- |
| 过程记录 | 日志     | 记录框架执行各阶段信息             |
| 数据记录 | RDR      | 记录框架执行状态数据，异常时自动导出 |
| 专项控制 | 内存复用 | 配置内存复用的开启与关闭           |
